# Smoke Evidence Artifact Schema (AT-010 / AT-011)

Smoke evidence generated by `scripts/run_smoke_and_capture_evidence.py` is standardized under:

- `docs/release/evidence/runs/<timestamp>/smoke_evidence.json`
- `docs/release/evidence/runs/<timestamp>/ubuntu_smoke_evidence.md` or `windows_smoke_evidence.md`
- `docs/release/evidence/runs/<timestamp>/*.log`

## JSON schema contract (required fields)

Top-level required fields:
- `schema_version`
- `run_id`
- `generated_at_utc`
- `target_os` (`ubuntu` or `windows`)
- `test_id` (`AT-011` or `AT-010`)
- `operator`
- `branch`
- `commit`
- `environment` (object)
- `commands` (non-empty array)
- `verdict` (`PASS` or `FAIL`)

Per-command required fields:
- `name`
- `command` (array of argv tokens)
- `started_at_utc`
- `completed_at_utc`
- `exit_code`
- `output_log`
- `expected_signatures`
- `matched_signatures`
- `status`

## Validation

- Programmatic validator: `run_smoke_and_capture_evidence.validate_evidence_payload(...)`
- Test coverage: `tests/test_smoke_evidence_generation.py`

## Contract notes

- Ubuntu mode executes smoke commands in this environment.
- Windows mode supports a `--contract-only` path so evidence shape/signature handling can be validated from non-Windows environments.
- AT-010/AT-011 may be upgraded from `partial` to `covered` when real per-OS run artifacts are attached under `docs/release/evidence/runs/`.
